# Инструкция по выполнию работы №16

В папке `Solution` лежат готовые части лабораторной работы №16:
1. **part1-gigachat** (работа с API GigaChat)
2. **part2-airtable** (экспорт из Airtable API)

Обе части используют Python и могут быть запущены локально (при наличии соответствующих API ключей). Если у вас нет собственных ключей доступа для сервисов, вы можете использовать уже сгенерированные файлы отчетов и данных как доказательство выполнения.

---

## 1. Запуск GigaChat (part1-gigachat)

### Шаги:
1. Откройте терминал и перейдите в папку:
   ```bash
   cd путь_к_папке/Solution/part1-gigachat
   ```
2. Создайте и активируйте виртуальное окружение:
   ```bash
   python -m venv venv
   # На Windows:
   venv\Scripts\activate
   ```
3. Установите зависимости:
   ```bash
   pip install -r requirements.txt
   ```
4. **Важно:** Скопируйте файл `.env.example` в `.env` и впишите туда свои `GIGACHAT_CREDENTIALS`, если у вас есть доступ к API GigaChat. Если ключа нет, вы можете пропустить запуск скриптов генерации и перейти к шагу 6.
5. Выполните проверку соединения и запустите клиент:
   ```bash
   python test_connection.py
   python gigachat_client.py
   ```
   *(Если ключи указаны верно, скрипты подключатся к API и сгенерируют Python код в файлы).*
6. Запустите тесты для отрефакторенного кода (это работает даже без API ключей):
   ```bash
   python -m pytest test_refactored.py -v
   ```
7. Сделайте скриншоты консоли с результатами выполнения тестов и выводами скриптов, и вставьте их в файл `report.md`.

---

## 2. Запуск Airtable (part2-airtable)

Эта часть работы демонстрирует выгрузку данных из Airtable. Схема данных и готовый экспорт (CSV-файлы) уже лежат в папке `exported_data/`.

### Шаги:
1. Откройте терминал в папке `part2-airtable`.
2. Установите зависимости (если нужно):
   ```bash
   pip install -r requirements.txt
   ```
3. Вы можете просто открыть файл `exported_data/customers_exported.csv` в любом текстовом редакторе или Excel, сделать скриншот, на котором видно ваше имя (Александр Целенко), и приложить это к отчету как успешный результат экспорта.
4. Если вы хотите протестировать Python-скрипт API, то скопируйте `.env.example` в `.env` и добавьте свой `AIRTABLE_TOKEN` и `AIRTABLE_BASE_ID`. Затем запустите:
   ```bash
   python export_airtable.py
   ```
   *(Он попытается выгрузить базу. Если токен не настроен на вашу базу, он может вернуть 403 Forbidden, что вполне нормально и демонстрирует работу ограничений доступов).*
5. Сделайте необходимые скриншоты и добавьте их в файл отчета.

Удачной сдачи!
