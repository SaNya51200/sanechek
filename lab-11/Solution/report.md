## 2. Теоретические сведения

### Подходы и инструменты (FastAPI)
- **FastAPI:** Высокопроизводительный веб-фреймворк для создания API на Python, построенный на базе Starlette (для веб-коммуникации) и Pydantic (для обработки данных). Поддерживает асинхронность (async/await) из коробки.
- **Pydantic-схемы:** Декларативное описание полей и ограничений сущностей. Валидация выполняется автоматически при разборе тела запроса или параметров.
- **Swagger UI:** Автоматически генерируемый интерактивный интерфейс спецификации API, доступный разработчикам по адресу `/docs`.

### Подходы и инструменты (Express)
- **Express.js:** Минималистичный и гибкий веб-фреймворк для Node.js приложений, предоставляющий базовый набор функций для маршрутизации и обработки HTTP-запросов.
- **Middleware (Промежуточное ПО):** Функции, имеющие доступ к объектам запроса (`req`), ответа (`res`) и следующей функции промежуточного ПО в цикле "запрос-ответ" (`next`). Применяются для логгирования, авторизации, валидации и обработки ошибок.
- **Joi:** Библиотека валидации схем данных для JavaScript. Позволяет описывать правила валидации программным путем (через цепочки методов) и проверять входящие объекты в рантайме.

### Часть 2: API менеджера задач (task-api)
Создан бэкенд на Node.js / Express для управления списком дел. Хранение организовано через локальный JSON-файл с использованием модуля `fs/promises` для асинхронного чтения и записи.

#### Фрагмент кода маршрутизации задач (`tasks.js`):
```javascript
// Создание новой задачи с валидацией Joi
router.post('/', checkTaskCreation, async (req, res, next) => {
  try {
    const { title, description, category, priority, dueDate } = req.body;
    const dbData = await readData();
    const createdTask = {
      id: await getNextId(), 
      uuid: uuidv4(), 
      title, 
      description: description || '',
      category: category || 'other', 
      priority: priority || 3, 
      dueDate: dueDate || null,
      completed: false, 
      createdAt: new Date().toISOString(), 
      updatedAt: new Date().toISOString()
    };
    dbData.tasks.push(createdTask);
    await writeData(dbData);
    res.status(201).json({ 
      success: true, 
      message: 'Задача добавлена в список', 
      data: createdTask 
    });
  } catch (error) { 
    next(error); 
  }
});
```

## 4. Контрольные вопросы

### Часть 1: FastAPI
1. **Преимущества Pydantic моделей:**  
   Pydantic обеспечивает декларативное описание ограничений данных непосредственно через типы Python (type hinting), что минимизирует количество ручных проверок и интегрирует автодополнение в редакторах кода.
2. **Как работает генерация документации в FastAPI:**  
   FastAPI на основе объявленных параметров путей, типов запросов и Pydantic-моделей автоматически строит структуру JSON, удовлетворяющую стандарту OpenAPI. Эта структура затем используется библиотеками Swagger UI или ReDoc для визуализации.
3. **Зачем нужна проверка уникальности ISBN:**  
   ISBN служит глобальным уникальным идентификатором печатного издания. Дублирование ISBN привело бы к логическим несоответствиям при выдаче и возврате книг в информационной системе.
4. **Статус-коды HTTP:**  
   - `200 OK` — для успешного извлечения данных.
   - `201 Created` — при успешном создании сущности (книги).
   - `204 No Content` — при успешном удалении без отправки тела ответа.
   - `400 Bad Request` — ошибки бизнес-логики (например, аренда уже занятой книги).
   - `404 Not Found` — если книга не найдена по указанному ID.
   - `409 Conflict` — при нарушении уникальности ISBN.

### Часть 2: Express
5. **Использованные Middleware:**  
   - `cors` — обработка CORS-запросов.
   - `helmet` — простановка защитных HTTP-заголовков.
   - `express-rate-limit` — лимитирование количества запросов с одного IP.
   - Кастомные: `checkTaskCreation`, `checkTaskModification` для валидации тела запроса и `checkTaskId` для валидации параметров пути.
   - `errorHandler` — централизованный отлов и обработка исключений.
6. **Различия валидации Joi и Pydantic:**  
   Joi описывает правила проверки через вызовы методов в JS-файле и валидирует данные в явном виде с помощью функции `.validate()`. Pydantic глубоко интегрирован в систему типов Python и декларативно обрабатывает входящие параметры без необходимости явного написания middleware-функций в хэндлерах.
7. **Плюсы файлового хранилища для лаб:**  
   Простота развертывания, отсутствие внешних зависимостей (СУБД) и персистентность данных между перезапусками сервера.
8. **Улучшение API для Production:**  
   Использование СУБД (например, PostgreSQL) вместо файлов, добавление аутентификации пользователей (JWT), ведение полноценных логов с помощью Winston/Pino и контейнеризация через Docker.
