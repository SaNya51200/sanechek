# Инструкция по запуску проектов

В папке `Solution` находятся две части лабораторной работы №15:
1. **part1-etl** (Пакетная обработка данных, Python, Pandas, SQLite)
2. **part2-kafka** (Потоковая обработка данных, Kafka, Python)

---

## 1. Запуск ETL-пайплайна (part1-etl)

### Шаги:
1. Откройте терминал и перейдите в папку проекта:
   ```bash
   cd путь_к_папке/Solution/part1-etl
   ```
2. Создайте виртуальное окружение и активируйте его (рекомендуется):
   ```bash
   python -m venv venv
   # На Windows:
   venv\Scripts\activate
   ```
3. Установите необходимые зависимости из файла `requirements.txt`:
   ```bash
   pip install -r requirements.txt
   ```
   *(Пакеты: pandas, matplotlib, seaborn).*
4. Запустите пайплайн:
   ```bash
   python etl_pipeline.py
   ```
5. В консоли вы увидите процесс извлечения, очистки и агрегации данных. Сделайте скриншот консоли для отчета.
6. Вы можете открыть сгенерированные графики в папке `report/graphs` (и вставить один из них в отчет) и проверить созданный файл базы данных `sales.db`.

---

## 2. Запуск Apache Kafka (part2-kafka)

Для выполнения второй части потребуется **Docker Desktop**.

### Шаги:
1. Откройте новый терминал и перейдите в папку Kafka:
   ```bash
   cd путь_к_папке/Solution/part2-kafka
   ```
2. Поднимите инфраструктуру Kafka с помощью docker-compose:
   ```bash
   docker-compose up -d
   ```
   *(Скачаются образы Zookeeper, Kafka и Kafka-UI. Подождите около 20-30 секунд, чтобы брокер успел инициализироваться).*
3. Активируйте виртуальное окружение Python и установите библиотеку `kafka-python`:
   ```bash
   pip install -r requirements.txt
   ```
4. Откройте **три** разных окна терминала (или вкладки) в этой же папке. Во всех окнах должно быть активировано виртуальное окружение.
   
   - **В первом терминале** запустите основного консьюмера (он будет ждать сообщений):
     ```bash
     python consumer.py
     ```
   - **Во втором терминале** (опционально) запустите оконного консьюмера:
     ```bash
     python windowed_consumer.py
     ```
   - **В третьем терминале** запустите продюсера, который сгенерирует поток заказов:
     ```bash
     python producer.py
     ```

5. Понаблюдайте за терминалами. Продюсер будет отправлять заказы с определенным интервалом, а консьюмеры в реальном времени будут их принимать и рассчитывать статистику (выручку, города, категории).
6. Сделайте скриншоты окон продюсера и консьюмера и вставьте их в файл `report.md`.
7. Вы также можете зайти в веб-интерфейс **Kafka UI** по адресу `http://localhost:8080`, чтобы наглядно посмотреть на топики и сообщения.
8. Чтобы остановить консьюмеров, нажмите `Ctrl+C`. Чтобы выключить Kafka, выполните:
   ```bash
   docker-compose down
   ```

Удачной работы!
