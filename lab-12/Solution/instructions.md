# Инструкция по сборке и запуску проектов

В папке `Solution` находятся два проекта:
1. **NotesApp** (Обязательная часть) - Android приложение на Kotlin (Jetpack Compose)
2. **NewsAggregator** (Часть на выбор) - Progressive Web App (PWA)

Ниже описаны шаги для запуска каждого из них.

---

## 1. Запуск Android-приложения (NotesApp)

Это полноценный Android-проект. Для его запуска вам потребуется установленная среда разработки **Android Studio**.

### Шаги:
1. Откройте **Android Studio**.
2. В стартовом окне выберите **Open** (или `File` -> `Open` в верхнем меню).
3. Укажите путь к папке `Solution/NotesApp` и нажмите `OK`.
4. Подождите, пока Android Studio синхронизирует проект с Gradle (это может занять несколько минут, внизу будет бежать полоса загрузки).
5. Нажмите на зелёный треугольник **Run 'app'** на верхней панели инструментов (или `Shift + F10`), предварительно выбрав подключенное Android-устройство (телефон по USB) или запущенный эмулятор (Virtual Device).
6. Приложение "Личные заметки" установится и откроется на вашем устройстве. Вы можете добавлять, редактировать и удалять заметки.

*Если возникнут ошибки с кэшем, вы можете очистить проект через `Build` -> `Clean Project` и затем `Build` -> `Rebuild Project`.*

---

## 2. Запуск PWA-приложения (NewsAggregator)

Это веб-приложение. Для его работы потребуется **Node.js** (для запуска локального сервера, чтобы работал Service Worker).

### Шаги:
1. Убедитесь, что у вас установлен Node.js (скачать можно с nodejs.org).
2. Откройте терминал (или командную строку / PowerShell).
3. Перейдите в папку с агрегатором новостей:
   ```bash
   cd путь_к_вашей_папке/Solution/NewsAggregator
   ```
4. Установите необходимые зависимости проекта (если они нужны, в данном проекте сервер может запускаться через встроенные скрипты или `npm install`):
   ```bash
   npm install
   ```
5. Запустите локальный сервер:
   ```bash
   npm start
   ```
   *(Если в package.json нет команды start, используйте `node server.js` или `node server/index.js` в зависимости от того, как устроен файл сервера)*.
6. Откройте браузер (рекомендуется Google Chrome или Edge) и перейдите по адресу:
   ```text
   http://localhost:3000
   ```
   *(Или другой порт, который будет указан в консоли после запуска).*
7. Вы увидите ваше PWA-приложение "Новости PWA". Чтобы проверить работу офлайн:
   - Нажмите `F12` в браузере (чтобы открыть Developer Tools).
   - Перейдите во вкладку `Network`.
   - Поставьте галочку `Offline`.
   - Обновите страницу. Вы должны увидеть интерфейс приложения с закэшированными данными и плашкой "Offline", вместо стандартной ошибки "Нет интернета" от браузера.
