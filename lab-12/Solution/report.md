## 2. Теоретические сведения

### Концепция Progressive Web Apps (PWA)
- **Service Worker:** Скрипт, запускаемый браузером в фоновом режиме, работающий как прокси-сервер между веб-приложением, сетью и локальным кэшем. Позволяет перехватывать сетевые запросы, отдавать кэшированные ресурсы при отсутствии соединения.
- **Web App Manifest (`manifest.json`):** JSON-файл метаданных, сообщающий браузеру о том, как приложение должно вести себя при установке на устройство пользователя (иконки, ориентация экрана, стартовый URL, цвета интерфейса).
- **IndexedDB:** Локальная транзакционная база данных, встроенная в браузер. Предназначена для структурированного хранения больших объемов данных (в отличие от localStorage).

### Концепция Android-разработки
- **Jetpack Compose:** Современный декларативный UI-фреймворк для Android. Описание интерфейса происходит через функции с аннотацией `@Composable`, которые автоматически перерисовываются при изменении их состояния (State).
- **СУБД Room:** Объектно-реляционная библиотека-обертка над SQLite от Google. Состоит из сущностей (`@Entity`), объектов доступа к данным (`@Dao`) и класса базы данных (`@Database`).
- **MVVM (Model-View-ViewModel):** Архитектурный паттерн, разделяющий бизнес-логику и UI. ViewModel управляет данными состояния (обычно через Flow/StateFlow), а View (Composable-экраны) наблюдает за ними и реагирует на изменения.

## 3. Практическая реализация

### Часть 1: PWA Новости (NewsAggregator)
Разработано легковесное новостное веб-приложение. Был написан сервис-воркер, кэширующий основные ресурсы, и логика сохранения постов во встроенную БД IndexedDB.

#### Фрагмент реализации Service Worker (`sw.js`):
```javascript
const CACHE_NAME = 'tselenko-news-cache-v1';
const ASSETS = [
  '/',
  '/index.html',
  '/src/js/app.js',
  '/src/js/db.js',
  '/manifest.json'
];

self.addEventListener('install', event => {
  event.waitUntil(
    caches.open(CACHE_NAME).then(cache => cache.addAll(ASSETS))
  );
});

self.addEventListener('fetch', event => {
  event.respondWith(
    caches.match(event.request)
      .then(cachedResponse => cachedResponse || fetch(event.request))
  );
});
```

## 4. Контрольные вопросы

### Часть 1: PWA
1. **Что такое Service Worker и жизненный цикл:**  
   Скрипт-прокси, работающий в отдельном потоке браузера. Этапы жизненного цикла: `register` (регистрация в основном JS), `install` (установка и кэширование статики), `activate` (активация и удаление старого кэша), и `fetch/idle` (обработка запросов).
2. **Как работает кэширование ресурсов:**  
   Service Worker перехватывает событие `fetch`. С помощью `caches.match()` проверяется наличие ресурса в кэше. Если ресурс найден — он возвращается мгновенно, иначе запрашивается из сети.
3. **Разница между LocalStorage и IndexedDB:**  
   LocalStorage — синхронное хранилище строк типа "ключ-значение" с лимитом около 5 МБ. IndexedDB — асинхронная нереляционная СУБД, поддерживающая индексы, транзакции и большие объемы сложных объектов.
4. **Условия для установки веб-приложения на экран:**  
   Наличие HTTPS (или localhost), зарегистрированного Service Worker с обработчиком `fetch`, и файла `manifest.json` с минимальным набором полей (иконки, name, start_url, display).

### Часть 2: Android
5. **Преимущества Jetpack Compose перед XML-разметкой:**  
   Сокращение объема кода, декларативный подход (UI описывается как функция от состояния), простая интеграция анимаций, отсутствие необходимости связывать элементы через `findViewById()`.
6. **Роль Room DB и его аннотации:**  
   Room упрощает работу с базой SQLite. `@Entity` задает структуру таблицы, `@Dao` описывает интерфейс SQL-запросов к БД, `@Database` объединяет сущности и предоставляет экземпляр подключения.
7. **Зачем нужен ViewModel и LiveData/StateFlow:**  
   ViewModel переживает переконфигурацию устройства (например, поворот экрана) и сохраняет актуальные данные. StateFlow предоставляет реактивный поток данных, на который подписывается UI для мгновенной перерисовки при изменениях.
8. **Что такое Coroutines в Kotlin:**  
   Это инструмент для написания асинхронного, неблокирующего кода в последовательном стиле. Используется в Android для выноса тяжелых операций (сеть, Room DB) из главного (UI) потока во избежание зависаний интерфейса.
