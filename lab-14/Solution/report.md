## 2. Теоретические сведения

### Реляционная СУБД PostgreSQL
- **Табличная структура:** Данные хранятся в связанных таблицах. Нормализация используется для устранения избыточности. Связи обеспечиваются внешними ключами (`FOREIGN KEY`).
- **Индексирование:** Создание дополнительных структур данных (чаще всего B-Tree) для быстрого поиска строк по определенным колонкам без полного сканирования таблицы (Sequential Scan).
- **План выполнения запроса (EXPLAIN):** Команда, показывающая шаги, которые оптимизатор базы данных предпримет для выполнения запроса (сканирование, хэширование, сортировка, фильтрация). `EXPLAIN ANALYZE` реально выполняет запрос и показывает фактическое время работы каждого узла.

### Документо-ориентированная NoSQL СУБД MongoDB
- **Документная модель:** Данные хранятся в формате BSON (двоичный JSON) в коллекциях документов. Поддерживаются вложенные структуры (массивы и подобъекты), что снижает необходимость дорогостоящих операций объединения (JOIN).
- **Денормализация:** Подход, при котором связанные данные хранятся непосредственно внутри одного документа. Это оптимизирует скорость чтения, но может усложнить обновление данных.
- **Индексы в MongoDB:** Как и в реляционных БД, используются для ускорения запросов. Доступны различные типы (одиночные, составные, мультикиви для массивов).

## 4. Контрольные вопросы

1. **Различия между реляционными (SQL) и NoSQL базами данных:**  
   SQL базы данных (например, PostgreSQL) имеют жесткую предопределенную схему, поддерживают ACID-транзакции и ориентированы на связи между нормализованными таблицами. NoSQL базы данных (например, MongoDB) предлагают гибкую схему (schemaless), поддерживают горизонтальное масштабирование и хранят денормализованные данные в виде документов, что ускоряет чтение.
2. **Как работает EXPLAIN в PostgreSQL:**  
   Позволяет увидеть план выполнения запроса, сгенерированный планировщиком. Он показывает используемые алгоритмы поиска (Seq Scan, Index Scan, Index Only Scan), способы объединения таблиц (Hash Join, Nested Loop) и оценку стоимости (cost) и объема строк.
3. **Что такое денормализация и ее плюсы/минусы:**  
   Это намеренное дублирование связанных данных в рамках одной сущности (например, встраивание адреса клиента в документ заказа). Плюсы: максимальная скорость чтения (все данные получаются одним запросом без JOIN-ов). Минусы: избыточность данных, риск рассогласованности при обновлении информации, повышенное потребление памяти.
4. **Виды индексов в базах данных:**  
   - B-Tree (стандартный сбалансированный индекс для поиска по диапазону и равенству).
   - Hash (для точного поиска по равенству).
   - GIN/GiST (для полнотекстового поиска и геометрических данных в PostgreSQL).
   - Составные индексы (по нескольким колонкам).
   - Мультиключевые индексы (в MongoDB для полей, являющихся массивами).
