# Инструкция по сборке и проверке проектов

В папке `Solution` находятся две части лабораторной работы №14:
1. **part1-postgresql** (Реляционная модель, SQL)
2. **part2-mongodb** (Документная модель, NoSQL)

Для обеих частей используется Docker и `docker-compose`. Убедитесь, что у вас установлен и запущен **Docker Desktop** (или Docker Engine).

---

## 1. Запуск и проверка PostgreSQL (part1-postgresql)

### Шаги:
1. Откройте терминал и перейдите в директорию PostgreSQL:
   ```bash
   cd путь_к_папке/Solution/part1-postgresql
   ```
2. Поднимите контейнер с базой данных:
   ```bash
   docker-compose up -d
   ```
   *(Docker скачает образ `postgres:15` и запустит его на порту 5432).*
3. Подождите несколько секунд, пока база инициализируется.
4. Подключитесь к контейнеру с помощью встроенного клиента `psql`:
   ```bash
   docker-compose exec postgres psql -U admin -d shop_analytics
   ```
5. Внутри `psql` вы можете выполнить команды из файла `schema.sql`, скопировав их из файла и вставив в терминал, либо выполнив файл, если он примонтирован.
   Для выполнения команд аналитики скопируйте запросы из `queries.sql` и вставьте их, чтобы увидеть результаты по категориям, топу клиентов (Александр Целенко) и динамике.
6. То же самое сделайте с командами из `index_analysis.sql`, чтобы проверить работу `EXPLAIN ANALYZE`.
7. Сделайте необходимые скриншоты для отчета.
8. Чтобы выйти из psql, введите `\q`. Для остановки контейнера введите:
   ```bash
   docker-compose down
   ```

## 2. Запуск и проверка MongoDB (part2-mongodb)

### Шаги:
1. Откройте новый терминал и перейдите в директорию MongoDB:
   ```bash
   cd путь_к_папке/Solution/part2-mongodb
   ```
2. Поднимите контейнер с базой данных:
   ```bash
   docker-compose up -d
   ```
   *(Docker запустит образ `mongo:6` на порту 27017).*
3. Подождите пару секунд для инициализации.
4. Подключитесь к базе данных через `mongosh`:
   ```bash
   docker-compose exec mongodb mongosh
   ```
5. В открывшейся консоли Mongo переключитесь на нужную БД:
   ```javascript
   use shop_analytics;
   ```
6. Скопируйте содержимое файла `init.js` и вставьте в консоль, чтобы создать коллекцию `orders` с данными.
7. Аналогичным образом копируйте и вставляйте код из файлов `queries.js`, `comparison.js` и `indexes.js`, чтобы выполнить базовые операции, агрегации и проверить работу индексов.
8. Сделайте скриншоты вывода консоли для отчета (например, вывод Aggregation Pipeline).
9. Чтобы выйти из `mongosh`, нажмите `Ctrl+C` два раза. Для остановки контейнера:
   ```bash
   docker-compose down
   ```

> [!TIP]
> Фактические логи выполнения в оригинальном примере сохранены в папке `verification`. Вы можете опираться на них для понимания, что именно должно появиться на экране. После снятия скриншотов вставьте их в файл `report.md` вместо пометок `*(Сюда скриншот)*`.
