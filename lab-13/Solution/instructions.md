# Инструкция по сборке и проверке проектов

В папке `Solution` находятся два проекта:
1. **part1-sast** (Статический анализ кода с использованием Bandit на Python/Flask)
2. **part2-sca** (Динамический анализ зависимостей и уязвимости OWASP на Node.js)

---

## 1. Запуск и проверка Python проекта (part1-sast)

Вам понадобится установленный **Python 3.8+**.

### Шаги:
1. Откройте терминал и перейдите в папку `part1-sast`:
   ```bash
   cd путь_к_вашей_папке/Solution/part1-sast
   ```
2. Создайте и активируйте виртуальное окружение (рекомендуется):
   ```bash
   python -m venv venv
   # На Windows:
   venv\Scripts\activate
   # На Mac/Linux:
   source venv/bin/activate
   ```
3. Установите зависимости (если их нет, установите Flask и Bandit вручную):
   ```bash
   pip install flask bandit
   ```
4. Для проведения статического анализа (SAST) выполните команду:
   ```bash
   bandit -r . -f txt -o bandit_report.txt
   ```
   *Вы можете сравнить полученный отчет с предыдущими результатами. В текущем коде большинство критических уязвимостей уже устранено.*
5. Чтобы запустить приложение, укажите API ключ (так как мы убрали хардкод):
   ```bash
   # На Windows PowerShell:
   $env:API_KEY="my_super_secret_key"
   python app.py
   
   # На Mac/Linux:
   export API_KEY="my_super_secret_key"
   python app.py
   ```
6. Откройте `http://127.0.0.1:5000` в браузере для работы с приложением.

---

## 2. Запуск и проверка Node.js проекта (part2-sca)

Вам понадобится установленный **Node.js**.

### Шаги:
1. Откройте терминал и перейдите в папку `part2-sca`:
   ```bash
   cd путь_к_вашей_папке/Solution/part2-sca
   ```
2. Установите зависимости пакета:
   ```bash
   npm install
   ```
3. Для проведения анализа зависимостей (SCA) на предмет уязвимостей (до исправления вы могли бы увидеть ошибки, но пакеты уже обновлены в `package.json`):
   ```bash
   npm audit
   ```
   *Если вы хотите симулировать наличие уязвимостей, можно вручную понизить версию `axios` или `ejs` в `package.json` и запустить команду снова. Для исправления уязвимостей используется команда `npm audit fix --force`.*
4. Установите переменную окружения `API_KEY` (т.к. хардкод был удален) и запустите сервер:
   ```bash
   # На Windows PowerShell:
   $env:API_KEY="my_super_secret_key"
   npm start
   
   # На Mac/Linux:
   export API_KEY="my_super_secret_key"
   npm start
   ```
   *(Если `npm start` не работает, запустите `node app.js`)*
5. Откройте `http://localhost:3000` в браузере для работы с приложением.
