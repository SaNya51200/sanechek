## 2. Теоретические сведения

### Анализ безопасности исходного кода (SAST)
- **SAST (Статическое тестирование):** Метод исследования защищенности ПО, при котором исходный код анализируется без реального выполнения программы. Позволяет обнаруживать потенциальные уязвимости (например, инъекции, слабые криптографические алгоритмы) на ранних стадиях жизненного цикла разработки.
- **Bandit:** Специализированный статический анализатор кода для поиска уязвимостей в проектах на языке Python.
- **SQL-инъекция (SQLi):** Уязвимость, возникающая при конкатенации пользовательского ввода в тело SQL-запроса без экранирования или параметризации.
- **Внедрение команд (Command Injection):** Уязвимость, позволяющая злоумышленнику выполнять произвольные системные команды на сервере через некорректную фильтрацию параметров запуска процессов (например, в модуле `subprocess`).

### Компонентный анализ зависимостей (SCA)
- **SCA (Анализ состава ПО):** Процесс идентификации и оценки рисков безопасности в сторонних компонентах и библиотеках с открытым исходным кодом, используемых в приложении.
- **npm audit:** Встроенная в менеджер пакетов npm утилита, которая сверяет дерево зависимостей проекта с базой данных известных уязвимостей (GitHub Advisory Database) и выдает отчет с рекомендациями по обновлению пакетов.
- **XSS (Cross-Site Scripting):** Уязвимость, позволяющая внедрять вредоносный клиентский скрипт в веб-страницу, просматриваемую другими пользователями. Защита включает экранирование вывода и использование Content Security Policy (CSP).

## 4. Контрольные вопросы

1. **Разница между SAST и DAST:**  
   SAST (Static Application Security Testing) анализирует исходный код без его запуска. Позволяет находить уязвимости на этапе написания. DAST (Dynamic Application Security Testing) тестирует работающее приложение снаружи (как черный ящик), симулируя атаки злоумышленника.
2. **Как работает параметризованный запрос к БД:**  
   Параметризация разделяет тело SQL-запроса (структуру команды) и передаваемые пользователем данные. Драйвер БД компилирует шаблон запроса заранее, а данные передаются отдельно, исключая возможность интерпретации ввода пользователя как части исполняемой SQL-команды.
3. **Что такое уязвимость XSS и способы защиты:**  
   XSS (межсайтовый скриптинг) — внедрение вредоносного JS-кода в страницу, которая отображается у других пользователей. Способы защиты: экранирование вывода (например, замена `<` на `&lt;`), валидация ввода, использование заголовков Content Security Policy (CSP) и флагов HttpOnly для Cookie.
4. **Назначение SCA-анализа и npm audit:**  
   SCA-анализ проверяет сторонние библиотеки (зависимости) на наличие известных уязвимостей. `npm audit` проверяет структуру `package-lock.json` по базе данных уязвимостей, классифицирует риски и помогает автоматически обновить проблемные зависимости.
